# Архитектура (целевая — полноценная система)

Один деплой-юнит: **`app/` — Next.js 16 App Router** (UI + API routes + фоновые джобы) поверх
**PostgreSQL 16**. Прод — VPS + cron/flock + Cloudflare Tunnel (как текущий crm-backend).

```
┌───────────────────────────────────────────────┐
│                app/ (Next.js 16)              │
│                                               │
│  UI (React 19, Tailwind 4, i18n RU/KK)        │
│   · офис: клиенты, сервис, биллинг, отчёты    │      ┌──────────────┐
│   · PWA техника: наряды, акт, подпись, фото   │─────▶│ PostgreSQL 16 │
│                                               │      │ (контейнер    │
│  API routes (JWT-кука + RBAC 7 ролей)         │◀─────│  crm-omnicomm)│
│                                               │      └──────────────┘
│  Джобы (cron-роуты + flock):                  │
│   · биллинг-начисления (посуточно)            │
│   · автоблокировка должников                  │
│   · e-mail-очередь расчётных документов       │
│   · sync-воркер Omnicomm                      │
└──────────────────┬────────────────────────────┘
                   ▼
        ┌─────────────────────┐     ┌─────────────────────┐

        │  блокировки ⇅        │     │ фид 1С:БУХ KZ → ЭСФ │
        └─────────────────────┘     └─────────────────────┘
```

## Домены (детали схемы — [DATA-MODEL.md](DATA-MODEL.md))

1. **Клиенты и договоры** — партнёр/контрагент (БИН/ИИН, ИИК/БИК/Кбе), соглашения, категории
   обслуживания, договоры госзакупа.
2. **Объекты мониторинга** — ТС (марка, госномер), привязка к клиенту и оборудованию.
3. **Оборудование и SIM** — серии/IMEI, размещения (склад/техник/клиент/поставщик/тестирование),
   состояния Новое/БУ, телематические состояния Активен/Консервация/Отключен + история (для биллинга).
4. **Сервис** — заявка ТО → заказ-наряд (график, периоды) → акт ТО (материалы, фото, SIM,
   снятое+установленное, подпись клиента) → движения оборудования и сдельные работы.
5. **Биллинг** — тарифы (иерархия объект>клиент>категория>умолчание), 3 типа начислений,
   аванс/кредит, НДС с датой действия, расчётные документы, рассылка.
6. **Сдельная ЗП** — категории, расценки, компенсации/удержания, ведомости.
7. **Интеграция** — учётки телематических серверов, журнал синхронизации, правила и журнал блокировок.
8. **Коммуникации** — тикеты, e-mail/Telegram/WhatsApp, звонки.

## Ключевые принципы

- **CRM — master** для платформы мониторинга: активация оборудования создаёт объект в Omnicomm,
  отключение — удаляет (двухэтапно); телеметрия забирается в кэш-снапшот (UI не дёргает Omnicomm).
- **История состояний** оборудования — единственный источник правды для посуточного биллинга.
- **Все списки выгружаются в Excel**; все печатные формы — двуязычные (KK/RU), формы РК (АВР Р-1, З-2).
- **i18n с первого экрана**, не ретрофитом.
- Легаси: `crm-backend/` (Express+SQLite, прод :3026) живёт до cutover; `liftplatform-omnicomm/` —
  источник проектных решений (SQL-миграции), не запускается.

## Интеграция с Omnicomm — правила (проверено на проде)

- Отчёты — `POST`, не GET.
- Топливо — децилитры, делить на 10.
- Дерево ТС — вложенное, флэттить.
- В units встречается баг ×10.
- Health-проба перед каждым забором; не запускать параллельные тяжёлые запросы к одной копии.
