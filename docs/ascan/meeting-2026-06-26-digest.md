## 1. Показанный функционал

### Продажи
- Заказ клиента: оборудование + услуги; **три порядка отгрузки**: при установке / без установки
  (карты тахографа) / до установки (муниципалы, монтаж со склада клиента).
- Счёт на оплату из заказа; «Отчёт состояния технического обслуживания» — контроль менеджера
  (по каким позициям заявка оформлена, работы выполнены).
- Реализация с печатной формой-расшифровкой по объектам обслуживания.

### Заявка на ТО (мост продажи → сервис)
- На основании заказа; связанные документы; объект = карточка ТС **или просто текст**
  (машина уточняется позже в наряде/акте).
- Помощник массового распределения «объекты × оборудование+услуги».

### Заказ-наряды (логист)
- **График заказ-нарядов**: drag-and-drop нераспределённых заявок на шкалу время × техники.
- Несколько исполнителей, несколько периодов (многодневные работы), автоподстановка адреса
  по прошлым выездам; печать / e-mail / мобильное приложение.

### Акт ТО (закрытие)
- Результат: выполнено / требуется доработка (→ новый наряд из акта).
- Оборудование и состояние («Активен» = старт абонплаты + автосоздание объекта в мониторинге).
- Работы (база сдельной ЗП), фотофиксация (места автозаполняются из вида работ),
  материалы (авто-списание расходников), регистрация SIM за оборудованием.
- Акты из мобильного приложения приходят «в подготовке» → офис проверяет и закрывает.

### Мобильное приложение техника (Android)
- Заказ-наряды и акты ТО с теми же вкладками; синхронизация в 1С; расписание работ.

### Тестирование оборудования
- Заказ на тестирование → монтаж → виртуальный склад; завершение = продажа или отказ (→ демонтаж).
- Отчёт «Товары на тестировании» с днями.

### Склад
- Размещения: склад / техник-исполнитель / клиент / поставщик / тестирование.
- Выдача технику только ручным «Перемещением товаров» (автоматики нет — боль дилера).
- Состояния Новое/БУ, подменный фонд; приём от клиента, передача поставщику (гарантия),
  отчёты с днями; расшифровка до документа-первоисточника.

### Сервисные операции с парком клиента
- Список оборудования клиента + операции: установка состояния (Активен/Отключен — абонплата и
  объект в мониторинге), перевод (другой клиент / организация / договор / объект), снятие с учёта.
- Регистрация оборудования клиента = миграция существующей абонбазы.
- Заявки техподдержки (тикеты) → решено офисом или → заявка ТО → наряд → акт.

### Сдельная ЗП (их доработка, в УТ нет)
- Категории исполнителей → расценки (по умолчанию / категория / конкретный исполнитель).
- Работы из актов + компенсации (амортизация, выезд) − удержания (штрафы).
- Месячный «Расчёт сдельного заработка» → ведомость → в бухгалтерию/ЗУП.

### SIM
- ТМЦ: склады / сотрудники / подрядчики / в оборудовании; Excel-импорт от оператора;
  перемещения всех направлений; отчёт остатков с тарифными планами.

### Абонплата (ядро)
- 3 типа начислений: по активному оборудованию (посуточно по истории вкл/выкл),
  подписки (фикс за период), разовые.
- **Иерархия тарифов**: категория → клиент → объект; скидки суммой до исчерпания.
- Схемы: авансовая (счёт в начале + акт в конце) / кредитная (всё в конце месяца).
- Массовое формирование расчётных документов + расшифровка абонплаты по объектам;
  e-mail-рассылка партиями, пакетная печать, 1С-ЭДО.
- Ведомость расчётов — долги/авансы.

### Интеграции с мониторингом
- 1С — master: активация → создание объекта в платформе, отключение → отключение объекта.
- РФ-версия: 10 платформ. **KZ-версия: только Wialon. Omnicomm — нет нигде.**

11. **Обмен с Omnicomm** — НЕТ.
12. **Автоблокировка должников** — НЕТ (руками в Omnicomm по отчёту).

14. Разные тарифы по технике — есть (иерархия).
