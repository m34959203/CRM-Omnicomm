# «Аскан: Мониторинг транспорта» — карта функционала (для паритет-анализа)

Источник: 112-страничный мануал ред. 2.5 (27.01.2025, `ascan-manual-2.5.txt`), ascan.su, TAdviser, Google Play.
Продукт: защищённая 1С-конфигурация ООО «Аскан» (Новосибирск), надстройка над УТ 11.5 / КА 2 / 1С:CRM 3.

## Модули

**Учёт оборудования клиентов** — разрезы: организация, клиент, контрагент, соглашение, абонентский
договор, объект мониторинга, номенклатура+характеристика+серия. Состояния: Активен (данные идут,
абонплата идёт) / Консервация (данных нет, абонплата идёт) / Отключен (удалён из СМ, не начисляется).
Документы: Регистрация / Снятие с учёта / Перевод (организация|клиент|договор|объект) / Установка
состояний. Отчёты: оборудование клиентов, движение, анализ активности.

**Техподдержка** — заявки (В работе/На обслуживании/Выполнена/Отклонена) → бизнес-процесс «Решение
проблемы» → решено удалённо / авто-создание заявок на ТО (по одной на объект) / отклонить.

**ТО (монтажи)** — Заявка ТО → Заказ-наряд → Акт ТО. Действия: Установка/Замена/Демонтаж/Диагностика/
Услуга; основания: Отгрузка по заказу / Отгружено ранее / Списание / Обслуживание (гарантия) /
Тестирование / Ответхранение — матрица определяет движения оборудования. График заказ-нарядов =
drag-and-drop по времени × исполнителям, группировка по логистам. Акт ТО — центральный документ:
регистрирует/снимает оборудование, списывает материалы по нормам расхода, ставит/снимает SIM,
фиксирует сдельные работы, результат (выполнено / доработка → новый наряд). Отчёты: просроченные
наряды, оборудование к установке, загруженность исполнителей (план/факт).

**Тестирование** — Заказ на тестирование → виртуальный склад тестирования → Завершение: Продажа
(→ заказ клиента) или Отказ (→ заявка на демонтаж). Отчёт с днями.

**Склад** — товары у исполнителей (сотрудники и подрядчики с личным складом) как разрез учёта;
4 вида перемещений склад↔исполнитель. БУ-учёт (нельзя продать/на тест), «Изменение состояний
товаров» (БУ↔Новый с ценой оприходования).

**Ремонт/замена** — Приём от клиента (долг перед клиентом, приход как БУ) / Выдача клиенту (можно
подменное) / Передача поставщику (виртуальный склад поставщика) / Получение от поставщика.
Отчёт «Продолжительность работы оборудования» (гарантийная аналитика).

**Сдельная оплата** — контракты (месяц/полмесяца), категории исполнителей, начисления Работа (из
актов) и Компенсация, удержания, товары в счёт оплаты. Тарифы: исполнитель > категория > умолчание.
«Расчёт сдельной оплаты» + ведомость (план/факт).

**SIM** — операторы, договоры, тарифные планы операторов; размещения: склад/сотрудник/подрядчик/
в оборудовании; Excel-импорт, перемещения, установка/снятие через акт (контроль max SIM в устройстве),
инвентаризация. Отчёты остатков и «SIM в оборудовании».

**Биллинг** — 3 способа: (а) по активному оборудованию — аванс за период по состоянию на начало,
факт посуточно; (б) подписки (посуточно, с корректировкой дат); (в) разовые. Тарификация: виды
тарифов (с валютой) → виды начислений → произвольные тарифы (объект > клиент > все клиенты,
«Не начислять» = нулевой) и/или тарифные планы (приоритетнее). Фикс-скидки до исчерпания.
Схемы: Авансовая (Заказ = счёт на аванс; формула: начисления + доначисление − скидки − предоплата) /
Кредитная (только акт). Массовое формирование + печать + e-mail (PDF: счёт, расшифровка абонплаты,
акт+СФ/УПД per-клиент). 6 аналитических отчётов.

**Интеграция с СМ** — документировано: Wialon Hosting/Local и Omnicomm Online (в РФ-версии; сайт
заявляет ещё Fort Monitor, ГлонассSoft, Gelios, GEO.RITM, AXENTA). Односторонняя, 1С — master:
аккаунты (Wialon), пользователи, объекты по сериям (имя, тип устройства, ID=IMEI/номер).
Консервация → очистка ID (Wialon) / «приём выключен» (Omnicomm). Отключение → двухэтапное удаление.
Сохранение/восстановление настроек объектов (.wlp / zip+xml). Начальная синхронизация с существующим
сервером + импорт объектов в 1С (парсинг марки/госномера из имени). **Блокирование при задолженности**:
правила по организации+категории, дни отсрочки отдельно для аванса/кредита, допустимая задолженность,
предупреждение (сообщение в СМ + email), журнал блокировок, ручная разблокировка «до даты».

**Оповещения** — email/SMS по событиям (расчётные документы, предупреждение о блокировке, пароли);
группы рассылок, шаблоны, регламентные задания.

**Мобильное приложение техника** (Android, 100+ установок): наряды к выполнению, оформление актов,
остатки товаров и SIM, начисления по ЗП.

## Ключевые сущности

Клиент(партнёр) ≠ Контрагент (юрлицо; холдинги); соглашения и договоры (Продажи / Абонплата / Ремонт);
категории сервисного обслуживания; объекты мониторинга (ТС: марка, госномер, VIN; или произвольные) +
шаблоны наименований; номенклатура с параметрами телематики + серии (IMEI, ID, PIN, телефоны SIM);
склады вкл. виртуальные; исполнители, установочные центры, логисты, контракты, категории; SIM-стек;
виды тарифов/начислений, тарифные планы, скидки, системы расчётов; телематические серверы (тип,
токен, корневой пользователь), учётки и пользователи СМ; нормы расхода материалов; причины диагностики.

## Бизнес-процессы end-to-end

1. Продажа+монтаж: Заказ → Заявка ТО → Наряд (график) → мобильное приложение → Акт (регистрация,
   SIM, материалы, сдельщина, синхронизация в СМ) → Реализация → «Активен» → биллинг.
2. Инцидент: тикет → БП → удалённо ИЛИ заявка ТО (диагностика) → наряд → акт → авто-закрытие тикета.
3. Биллинг-цикл: начало месяца — авансовые счета → рассылка PDF → конец месяца — акты (посуточный
   факт) → при долге: предупреждение → блокировка в СМ → разблокировка после оплаты.
4. Тест-драйв: заказ на тест → монтаж → продажа или демонтаж.
5. Ремонт: приём от клиента (БУ) → поставщику → получение → выдача/замена по гарантии.
6. ЗП: акты + компенсации − удержания → расчёт за период → ведомость.
