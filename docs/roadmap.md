# Roadmap

Полный план с обоснованием: [ASCAN-PARITY-PLAN.md](ASCAN-PARITY-PLAN.md).
Цель — полноценная система уровня «Аскан: МТ» с козырями, которых у Аскан нет:
Omnicomm-нативность (включая телеметрию), автоблокировка должников, РК-локализация.

## Этап 0 — Фундамент (~1 нед) — СДЕЛАН
- [x] `app/`: Next.js 16 App Router + TS + Tailwind 4, PostgreSQL 16 (отдельный контейнер)
- [x] Миграции по [DATA-MODEL.md](DATA-MODEL.md), seed
- [x] Auth (JWT httpOnly-кука) + RBAC 7 ролей + аудит-лог
- [x] Каркас i18n RU/KK
- [x] CRUD: клиенты (БИН/ИИН, ИИК/БИК/Кбе, юр. форма, признак НДС), объекты, оборудование, SIM, склады
- [x] Сквозной паттерн «список → Excel»
- [x] Миграция данных прода SQLite → PG

## Этап 1 — Интеграция Omnicomm Online (~1.5 нед) — СДЕЛАН
- [x] Справочник телематических серверов + health-проба
- [x] CRM-master: активация → объект в Omnicomm; консервация; двухэтапное удаление
- [x] Импорт существующей абонбазы (парсинг марки/госномера)
- [x] Регистр истории состояний (фундамент посуточного биллинга)
- [x] Автоблокировка должников (правила, предупреждение, журнал)
- [x] Резерв настроек объекта перед удалением

## Этап 2 — Биллинг v2 (~2 нед) — СДЕЛАН
- [x] 3 типа начислений: посуточно по активности / подписки / разовые
- [x] Иерархия тарифов: объект > клиент > категория > умолчание; тарифные планы; скидки суммой
- [x] Схемы: авансовая / кредитная; НДС 16% как параметр с датой действия
- [x] Массовое формирование расчётных документов + двуязычные PDF (счёт, расшифровка, АВР Р-1, З-2)
- [x] Ведомость расчётов → автоблокировка
- [x] Выгрузка в 1С:Бухгалтерию KZ (→ ЭСФ/ЭАВР в ИС ЭСФ); сценарий бюджетников
- E-mail-рассылка расчётных документов очередью — в бэклоге (очередь уведомлений готова, этап 6)

**▶ Веха: демо заказчику после этапов 0–2 (~4–5 недель).**

## Этап 3 — Сервисный контур (~2 нед) — СДЕЛАН

- [x] Автоперемещение оборудования на техника; командировки в наряде

## Этап 4 — PWA техника (~1.5 нед) — СДЕЛАН

## Этап 6 — Техподдержка и коммуникации (~1 нед) — СДЕЛАН
- [x] Тикеты → авто-заявки ТО → авто-закрытие; e-mail/Telegram/WhatsApp (Baileys); журнал звонков

## Этап 7 — Отчёты и полировка (~1 нед) — СДЕЛАН
- [x] Сводный отчёт по оборудованию во всех размещениях (`/reports/equipment`: группы → расшифровка до единицы, Excel обоих уровней)
- [x] Отчёты «с днями»: тестирование / принято от клиентов / у поставщика (`/reports/days`)
- [x] Загруженность исполнителей план/факт (`/reports/workload`); анализ активности оборудования (`/reports/activity`)
- [x] Дашборд руководителя (`/dashboard`: заявки/просрочка, наряды, тикеты, биллинг месяца, топ должников, ошибки синхронизации, занятость техников)
- [x] Карточка клиента (`/clients/[id]`): реквизиты, парк с телематикой; установка состояния / перевод другому клиенту / перенос на объект — ESH-переходы с непрерывностью биллинга
- [x] Добивка казахской локализации + переключатель языка RU/KK; суммы прописью KK/RU
