## Ключевые выводы для parity CRM-Omnicomm

### Биллинг (приоритет)
- **Тарифная модель двухуровневая:** тарифный план (по «виду тарифа» + «категории обслуживания»,
  помесячные ставки в периодическом регистре по «видам начисления») **+** произвольные тарифы
  (документ «Установка тарифов», пообъектная ставка или «Не начислять», с периодом действия).
- **Прогон биллинга:** обработка «Формирование и рассылка расчётных документов» —
  вариант расчёта **Аванс** (по заказам) / **Факт** (по актам), период-месяц, богатые отборы
  (организация/клиент/менеджер/категория/сегмент/регион/состояние отправки), пакетное
  «Сформировать документы» + «Подготовить и отправить письма» (ЭДО), состояние ЭД, подписант.
- **Отчёты:** «Анализ начислений» (расшифровка аванс/факт/разница по клиенту и объекту),
  «Ведомость расчётов и авансовых платежей» (полная дебиторка: нач./кон. остаток, увеличение/
  уменьшение долга, сальдо, аванс, к оплате).

### Интеграция с системами мониторинга (проверено отдельно по запросу)
- Учётные записи систем мониторинга поддерживают серверы **Omnicomm Online**, Wialon Hosting,
  Wialon Local. Карточка аккаунта: «Синхронизировать», список объектов мониторинга, пользователи.
- **Связка биллинг ↔ платформа:** флаг «Управлять блокированием» + режим
  **«блокировать при задолженности автоматически»** → неоплата абонплаты влечёт автоблокировку
  объектов клиента в Omnicomm Online. Журнал «Ошибки синхронизации» фиксирует неуспешные попытки
  (пример: «Не удалось блокировать учётную запись. Код ошибки: 7»). Это критичный сценарий для репликации.

### Сервис
- Контур: Заявка на ТО → Заказ-наряд (+ диспетчерский «График заказов-нарядов» по технику/времени)
  → Акт ТО. Акт ТО содержит вкладки: Основное, Исполнители, Результат работ (осмотр/диагностика/
  причины), **Оборудование/услуги** (номенклатура+серия+состояние, привязка к объекту),
  **Работы** (сдельно по исполнителям), **Материалы** (списание по нормам).
- Тестирование оборудования: «Заказы на тестирование» / «Завершения тестирований», склад
  «Оборудование на тестировании» (оборудование на пробу клиенту).

**Ключевые отличия роли техника (для parity):**
- **Урезанный UI:** левое меню только «Главное» + «НСИ и администрирование»; вся работа — в экране-лаунчере
  **«Рабочее место техника-установщика»** (Документы: Заказы-наряды; Отчёты: Расписание работ исполнителя,
  **SIM - карты**, Ведомость по сдельному заработку, Товары и оборудование БУ на складах).
- **Заказы-наряды** — упрощённый список только своих открытых нарядов, кнопка «+» в колонке «Акт» создаёт акт.
- **Акт ТО — ДРУГАЯ форма:** не вкладки, а единый прокручиваемый лист со сворачиваемыми разделами. Добавлены
  полевые элементы: **«Добавить подпись клиента»**, «Подписант от клиента», «Несколько исполнителей»; печать
  **«Заказ-наряд с подписями»**.
- **Оборудование в акте техника — шире:** учитывает и установленную, и **снятую** номенклатуру/серию/состояние
  (сценарий замены прибора: снял старый → поставил новый, с фиксацией серий обоих).
- **Фотофиксация** = вложения **«Файлы»** к акту (не отдельная вкладка; доступ после записи акта).
- **SIM** = скрытый контекстный подраздел «SIM-карты» в акте **+** отдельный отчёт **«Остатки SIM-карт»**
  (группировка Склад → Сотрудник → Оборудование, фильтр «Тип размещения»). В демо-данных SIM-записей нет.

**Вывод:** интерфейс техника — отдельный «полевой» сценарий поверх тех же документов. Для CRM-Omnicomm заложить:
роль/АРМ техника с урезанным меню, форму акта с электронной подписью клиента, учёт снятого оборудования при
замене, фотофиксацию (вложения) и учёт SIM с движением по местам хранения.
